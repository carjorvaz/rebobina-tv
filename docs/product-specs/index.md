# Product Specs

Current product target:

- `catch-up-handoff-gallery.md` - sofa-friendly catch-up browsing and provider
  handoff validation.
- `metadata-adapter-boundary.md` - safe metadata-only adapter contract.
- `provider-evidence/` - redacted route evidence observed on device.
- `provider-evidence.schema.json` - expected structure for provider evidence.

Future product specs should include:

- User problem and target device.
- In-scope and out-of-scope behavior.
- Acceptance criteria.
- Validation evidence required before claiming support.
